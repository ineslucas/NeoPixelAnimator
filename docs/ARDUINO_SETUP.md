# Arduino setup

Everything you need to take the code generated by NeoPixel Planner and run it on real hardware.

## What you'll need

- A microcontroller. Any of these work:
  - Arduino Uno / Nano / Mega
  - ESP32 or ESP8266
  - Raspberry Pi Pico (with Arduino core)
  - Adafruit Feather, Trinket, ItsyBitsy
- A NeoPixel-compatible LED strip:
  - WS2812B (most common, sold as "NeoPixel" by Adafruit)
  - SK6812 (RGBW variant)
  - WS2811 (older, often in 12V "pixel string" form)
- Hookup wire, a 5V power supply rated for your strip, and ideally a 1000 µF capacitor + 470 Ω resistor (see Adafruit's NeoPixel Überguide for why).

## 1. Install the library

In the Arduino IDE:

1. **Tools → Manage Libraries…**
2. Search **Adafruit NeoPixel**.
3. Click **Install** on the latest version.

## 2. Wire the strip

Three connections:

| Strip pin    | Connect to              |
|--------------|-------------------------|
| **+5V (VCC)** | 5V power supply (NOT the Arduino's 5V if you have more than ~10 LEDs) |
| **GND**       | Ground (also connect to Arduino GND) |
| **DIN (Data In)** | Arduino digital pin specified in the sketch (`LED_PIN`, default 6) |

Recommended: a 470 Ω resistor between the Arduino data pin and the strip's data input, and a 1000 µF capacitor across the +5V/GND of the strip near the first LED. Protects the first LED from voltage spikes.

> **Power note.** Each WS2812B can draw up to ~60 mA at full white. A 60-LED strip can pull 3.6 A. Don't power long strips from your Arduino's USB. Use a dedicated 5V supply.

## 3. Flash the sketch

1. In NeoPixel Planner's **Export** step, click **Download .ino** (or **Copy code** and paste into a new sketch).
2. Open the `.ino` file in the Arduino IDE.
3. **Tools → Board** → select your microcontroller.
4. **Tools → Port** → select the USB port it's connected to.
5. Click the **Upload** button (right-arrow icon).

If the strip lights up and runs your animation, you're done.

## Troubleshooting

**Nothing lights up.**

- Check `LED_PIN` matches the pin the strip's DIN is connected to.
- Check polarity on the +5V/GND. The strip has a directional arrow. Data flows in one direction only. The first LED is on the *DIN* side.
- Check the strip is getting power independently (a multimeter on +5V/GND should read 5V).

**Only the first LED lights up.**

- The strip's data signal is being received only by LED #0. Data line isn't reaching LED #1. Check the solder joints between the first and second LED.

**LEDs flicker erratically.**

- Power supply isn't beefy enough. Try a higher-current 5V brick.
- Add the 1000 µF capacitor across +5V/GND if you haven't.
- For long runs (>1 m of wire between Arduino and strip), use shielded cable or a level shifter (74AHCT125). Arduino's 5V logic can be marginal at distance.

**Wrong colors (e.g., blue is showing as red).**

- Your strip is a different ordering than NEO_GRB. In the **Export** step, change **LED type** to RGB or GRBW and re-export.

**`Adafruit_NeoPixel.h: No such file or directory`.**

- Library not installed. Re-do step 1 above.

## Going further

- **Power injection.** Long strips need 5V/GND injected at multiple points along the strip. The LEDs at the far end won't get enough voltage otherwise.
- **Level shifters.** 3.3V boards (ESP32, RP2040) sometimes need their data line boosted to 5V for reliable signaling. A 74AHCT125 chip does this cheaply.
- **Cool projects to remix the generated code into:** light-up costumes, signage, status indicators, ambient room lighting, art installations.
